# KAZAN-GAICA

Проект для разработки бота под соревнование GAICA.

Сейчас репозиторий содержит базовую структуру проекта, которую команда может постепенно заполнять своей логикой: сетевым клиентом, парсером состояния игры, стратегией и тактикой бота.

## Структура проекта

- `main.py` - точка входа для запуска бота.
- `smurf_bot/` - пакет для основной логики бота.
- `smurf_bot/client.py` - заготовка под TCP-клиент и обмен JSON-сообщениями.
- `smurf_bot/controller.py` - заготовка под главный игровой цикл и принятие решений.
- `smurf_bot/game_state.py` - заготовка под разбор состояния матча и снапшотов.
- `smurf_bot/geometry.py` - заготовка под геометрию, расстояния и проверки столкновений.
- `smurf_bot/models.py` - заготовка под модели данных и структуры команд.
- `smurf_bot/strategy.py` - заготовка под стратегию и эвристики.

## Идея архитектуры

Бота удобно разделить на несколько независимых частей:

1. Сетевой слой: подключение к раннеру, отправка `register`, чтение `hello`, `round_start`, `tick`, `round_end`.
2. Слой состояния: преобразование входного JSON в удобные Python-структуры.
3. Геометрия и проверки: дистанции, направления, line-of-sight, безопасное движение, анализ края карты.
4. Стратегия: выбор режима поведения.
5. Тактика на тик: формирование конкретной команды.

## Предлагаемый план разработки

1. Реализовать минимальный TCP-клиент по правилам соревнования.
2. Добавить логирование входящих сообщений для проверки формата JSON.
3. Собрать внутренние модели игрока, врага, препятствий, оружия и пола.
4. Сделать простого rule-based бота.
5. После этого усиливать эвристики и тестировать локально через раннер.

## Запуск

Предполагаемый формат запуска:

```bash
python main.py <host> <port>
```

## Заметки

- В репозитории пока только каркас.
- Основную игровую логику команда может реализовать самостоятельно по модулям.
- Перед отправкой на платформу стоит проверить структуру архива и локальный запуск через раннер соревнования.
